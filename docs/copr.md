# Fedora COPR Publishing

WSF can be published to Fedora COPR from a source RPM generated by the
repository packaging workflow. COPR then builds Fedora RPM repositories that
users can enable with `dnf copr enable`.

This is separate from Fedora official repositories. COPR is the first Fedora
distribution target; official Fedora packaging requires a later package review.

## One-Time COPR Setup

1. Create or use a Fedora account:

   ```text
   https://accounts.fedoraproject.org
   ```

2. Log in to COPR:

   ```text
   https://copr.fedorainfracloud.org
   ```

3. Open the COPR API page and copy the generated token block:

   ```text
   https://copr.fedorainfracloud.org/api/
   ```

   It looks like:

   ```ini
   [copr-cli]
   login = ...
   username = ...
   token = ...
   copr_url = https://copr.fedorainfracloud.org
   # expiration date: ...
   ```

   Do not commit this token. COPR tokens expire, so expect to rotate it.

4. Create the COPR project once. You can do this from the COPR web UI, or from
   any machine/container with `copr-cli` configured:

   ```bash
   copr-cli create wayland-scroll-factor \
     --chroot fedora-rawhide-x86_64 \
     --description "Touchpad scroll and gesture tuning for Wayland (GNOME and Hyprland)" \
     --instructions "Install with: sudo dnf copr enable <username>/wayland-scroll-factor && sudo dnf install wayland-scroll-factor" \
     --appstream on \
     --follow-fedora-branching on
   ```

   Add stable Fedora chroots from the COPR web UI after project creation. COPR
   chroot names change as Fedora releases move, so the web UI is the safest
   place to select the currently supported Fedora versions.

## GitHub Secret

Add a repository secret named `COPR_CONFIG` containing the full token block from
the COPR API page.

GitHub path:

```text
Repository -> Settings -> Secrets and variables -> Actions -> New repository secret
```

Name:

```text
COPR_CONFIG
```

Value:

```ini
[copr-cli]
login = ...
username = ...
token = ...
copr_url = https://copr.fedorainfracloud.org
# expiration date: ...
```

## Publish From GitHub Actions

Use the manual workflow:

```text
Actions -> Publish COPR -> Run workflow
```

Inputs:

- `project`: usually `wayland-scroll-factor`; use `username/wayland-scroll-factor`
  if publishing outside the authenticated user's namespace.
- `nowait`: keep enabled for normal publishing. Disable it only when you want
  the workflow to wait for COPR build completion.

The workflow:

1. Builds RPM/SRPM artifacts in a Fedora container.
2. Uploads those artifacts to the GitHub Actions run.
3. Submits the generated `.src.rpm` to COPR with `copr-cli build`.

## Manual Local Build

When debugging locally:

```bash
mkdir -p package-artifacts/rpm
WSF_PACKAGE_OUTDIR="$PWD/package-artifacts/rpm" scripts/test-packages.sh rpm
copr-cli build wayland-scroll-factor package-artifacts/rpm/*.src.rpm
```

This requires `~/.config/copr` to contain your COPR token.

## User Installation Command

Once the COPR build succeeds, Fedora users can install with:

```bash
sudo dnf copr enable <username>/wayland-scroll-factor
sudo dnf install wayland-scroll-factor
```

Replace `<username>` with the COPR/Fedora username that owns the project.
